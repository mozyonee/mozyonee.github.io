Tutorial 8: PolyAnalyst Scheduler

 

Brief Description: The Scheduler is a tool that provides batch process data mining in Polyanalyst.  The Scheduler allows more advanced users to create powerful data mining scenarios based on a simple but flexible proprietary scripting language of PolyAnalyst.  The Scheduler can be used for running specified exploration techniques on specified data sets in specified order at specified times, and output the results of exploration in specified files.  Also, the scripts for the Scheduler can be saved and used as templates for future data mining tasks.

The script we are about to create will contain similar task that were done in tutorial three (analysis of factors influencing gas consumption).  If you have not done tutorial three, you might want to do it before this tutorial to help you better understand the data mining tasks we perform.  However, the scripting code can still be easily understood.

Note:  If you want to see the Polyanalyst Scheduler perform without learning how to create the script you can move to Step 3 using “AutompgS.sch” as the file to open instead of the file “Autompg_Script.txt”. 

Now let us create a simple script to run in the Polyanalyst Scheduler. 

Step 1: Opening Notepad and Creating a Script File

Scripting files are made using programs that can save files as text documents (e.g. Notepad).  The Scheduler files can have the extension “.sch” or “.txt”.

 

1.      Open Notepad. 

2.      Select File | Save from the Notepad menu bar.  From the Save as dialog box name the file “Autompg_Script” and save the file in the PolyAnalyst Examples folder.

 

Default Location: C:\Program Files\Megaputer Intelligence\PolyAnalyst 4.1\Examples\

 

Note: Using the copy and paste functions (from Word to Notepad) will cause an error   because the PolyAnalyst Scheduler cannot interpret the pasted format and you will receive an error.  However, you can cut and paste from one Notepad file to another.

 

Now that our file has been created we can start to write the script commands that will be carried out in PolyAnalyst by the Scheduler.

Step 2: Writing Script Commands

First, we are going to create a comment for future reference of the files purpose.  To create a comment you must start a line with the single quotation mark (‘).

1.      In the first line of the Autompg_Script text file type:

‘Example script for scheduler

 

2.      Push the Enter key twice to space out the text to make it easy to read and also because Script Commands must be on their own line.

 

Now we can open the data to create a project.  The syntax for creating a file is:

create project from <”file”>

3.      Type the following line:

create project from “Autompg.csv”

 

4.      Push the Enter key twice.

 

Note:  If you would like to indent you script code to make it more recognizable, use spaces rather than tabs.  The Scheduler will cause an error if it encounters a tab, unless it is after a single quote as part of a comment. 

 

In our example the data is in the same folder as the script we are creating so we do not need to use the file directory name.  However, if the files were in different locations you would have to enter the path of the file.

The World dataset is created and now we are going to create a rule.  The syntax for creating a rule is:

create rule <”name of rule”> as <”expression”>

5.      Type the following line:

create rule “Age1” as “83 – Year”

 

6.      Push the Enter key twice.

 

Note: The command keywords (e.g. “create rule”) are case insensitive, but the projects collection and objects names are case sensitive.

Now that the rule has been created we are ready to apply it to the World dataset.  The syntax for applying a rule is:

apply rule <“name”> to <“dataset”>

7.      Type the following line:

apply rule “Age1” to “World”

 

8.      Push the Enter key twice.

 

Now let us create an exploration dataset without the attributes Year and model.  Recall from tutorial three (analysis of factors influencing gas consumption) that we found these attributes to not provide much insight in any further analysis.  The script commands recognize columns as a mask using digits ‘1’ and ‘0’ as places holders.  In our case, we have 10 placeholders in the World dataset for our 10 attributes (1111111111).  The digits represent their respective attribute by order of left to right.  Column one being the first digit, column two being the second digit and so on.  The digit ‘1’ represent the column as being existent and the digit ‘0’ represent the column as being not included.

The syntax for creating a new dataset is:

create dataset <”dataset name”> from <”name of dataset”> columns <”mask”>

9.      Type the following line:

create dataset “Explored” from “World” columns “1111110101”

 

10.  Push the Enter key twice.

 

About the script command:  The attribute Year is the eighth column in the World dataset and, the attribute model is the ninth column in the World dataset.  We excluded these attributes in the Explored dataset by entering a ‘0’ in their respective placeholders.

Now let us run Summary Statistics on the Explored dataset.  The syntax for running SS is:

Start SS on <”dataset”> name <”name”>

11.  Type the following line:

start SS on “Explored”

 

12.  Push the Enter key twice.

 

Next we might want to make a learning dataset for the Find Laws exploration engine.  We are going to take a random sample from the Explored dataset.  The syntax for creating a new dataset using by taking a random sample is:

create dataset <”dataset name”> from <”name”> add random <n>%

 

13.  Type the following line:

create dataset “learn” from “Explored” add random 50%

 

14.  Push the Enter key twice.

 

Now we want to run Find Laws on the learn dataset with the target attribute as mpg, which is the first column in the dataset so the target attribute is denoted as ‘1’ for the target id, and a time limit of one minute will be used to stop the Find Laws algorithm.  The syntax for running FL is:

start FL on <”dataset”> name <”name”> target <column id> limit <time limit>

 

15.  Type the following line:

start FL on “learn” name “FL_Learn” target 1 limit 1

 

16.  Push the Enter key twice.

 

Next we want to save our project, but there is one more step we must do first.  We want the FL exploration engine to be done before we save the project.  This would also be the case if you wanted to apply the FL model (as a rule) to a dataset because you have to wait until the FL exploration engine completed. 

 

17.  Type the following line to hold the program until the FL exploration engine is completed:

wait for completion with time limit 3.2

 

18.  Push the Enter key twice.

 

About the script:  This command will wait 3.2 minutes or until the FL exploration engine finishes depending upon which comes first.  Now let us complete the script by saving the project

19.  Type the following line to save the project:

save project to “autompg_script.prj”

 

Step 3: Running the Scheduler in PolyAnalyst

Our script file is complete.  Now we are going to run it using PolyAnalyst Scheduler.

 

1.      Open PolyAnalyst.  When the Start up dialog box appears, select Cancel. If you already have PolyAnalyst open you need to close any open projects because the Scheduler option is not available when any projects are open.

2.      From the menu bar select Project | Scheduler | Schedule.

 

 

 

3.      In the Select the file with PolyAnalyst Scheduler commands dialog box we need to change the Files of types drop down box to be “Text Files (*.txt)”.  Next we need to select our file “Autompg_script.txt”.

 

Default Location: C:\Program Files\Megaputer Intelligence\PolyAnalyst 4.1\Examples\

 

 

4.      Once you find the file “Autompg_script.txt” double click on it.

 

Now you can watch the PolyAnalyst Scheduler create your project from the commands you scripted.  The Scheduler will start off by creating the project from the “autompg.cvs” dataset, then create the “Age1” rule and apply it to the World dataset.  Next, the Scheduler will create the Explored dataset and run Summary Statistics on it.  Lastly, the Scheduler will create the Learn dataset and run Find Laws on the Learn dataset with the target attribute as mpg.  When the Scheduler is complete you can close PolyAnalyst.  For more information on the Scheduler and script commands or for a copy of this tutorial’s script, refer to Appendix F.  To view a copy of the default script file open Notepad and then from Notepad open the file “AutompgS.sch”.
