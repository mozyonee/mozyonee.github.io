Tutorial 1: Simple and the Most Exact Dependence

Brief description: This tutorial illustrates how PolyAnalyst can be used for performing an automated discovery of a formula that relates two attributes.  The data consists of two attributes (variables): x and y. Pairs of values of these attributes were generated artificially according to the following simple formula:

y = (2 *x*x+30000)/(x+300)

Step 1: Creating the Project, Importing Data into PolyAnalyst:

1.      From within the PolyAnalyst program, select from the main file menu Project | New.

2.      The Select data source type dialog window appears.  PolyAnalyst provides six different file types to choose from for importing, allowing PolyAnalyst to easily work with several existing database types. 

3.      Notice that the first option Flat File (comma separated values) is selected.  Comma Separated Value files end with the default file extension *.csv.  You will be using this file type in this tutorial.

4.      Click Next to continue.

5.      The next step of the import wizard requests you to enter the file name and its location so that PolyAnalyst may import it.  Click Browse.

6.       The open dialog box appears.  You should be in the default folder of Examples.  This directory is located by default at:

C:\Program Files\Megaputer Intelligence\PolyAnalyst 4.1\Examples\

7.  Select the file named 1simple.csv.

About the CSV File:  The 1SIMPLE.CSV data file contains the corresponding values of (x, y) pairs with x ranging from 1 to 591 incrementing by 1.  If you are curious what the data in CSV format looks like, you can view this file with the help of any text editor.  You can use Microsoft Excel or most other database applications to create a CSV file.

8.      Click Finish.

9.      The Edit Project dialog box appears.

10.   For future reference, name the project Simple and Exact Dependence in the edit box after Project Name:

In the CSV file, the corresponding values of the (x,y) pairs are separated into two columns or fields.  PolyAnalyst determines these columns and intelligently suggests what data type to assign to each field.  You can accept the default data types, or assign a data type to each field by clicking the text in the window under Type.  If you click “number”, a drop down menu appears listing other data types you may select.  In this tutorial, both fields are numerical.

11.  Click OK.

Upon clicking OK, PolyAnalyst imports the data from the CSV  file.  Data is imported into a data set, or container for data.  The first import results in the creation of the World dataset.  This dataset contains all the attributes, or fields, from the CSV file.  You can quickly view the World data set by clicking on the + icon next to the word Datasets in the Workspace panel on the left, then double- clicking on the World dataset.

Step 2: Using a Data Exploration Algorithm (Find Laws)

When researching data one of the common goals is to determine the formula expressing the dependence of the y variable on the x variable (if it exists).  There are several basic statistical applications that can formulate this basic linear regression equation.  PolyAnalyst uses several different approaches to understanding relationships between variables.  One of these approaches is the Find Laws data exploration algorithm.

There are several ways to execute algorithms.  Most functions are available through the main file menu or from the toolbar.  PolyAnalyst also uses extensive context menus reached by right clicking on certain areas of the PolyAnalyst interface.

1.      Right click on the World dataset.  Select Explore | Find Laws …(FL).

2.      The Find Laws in Dataset (World) dialog appears.

3.  To find the relationship between two variables, PolyAnalyst needs the user to determine the dependant and independent variables.  The dependant variable is also referred to as the target attribute, or target.  To select the y attribute as the target variable, double click on it under the Dataset Fields area.  Notice that the y attribute is now highlighted in red.

3.      Click OK.  PolyAnalyst launches the Find Laws data exploration process.

About The Find Laws Exploration Process:  Three actions occur on execution of the algorithm with viewable changes in both the Workspace panel and the View panel.  First, the process report is created in the Workspace panel for the duration of the exploration time period.  This serves as an in- progress report of the exploration results.  Secondly, two new rules are generated within the Workspace panel, FL_y and FL_y_Ex.  The prefix FL in the rule names stands for Find Laws.  To view the rules in PolyAnalyst’s Rule Assistant, click on the rule’s name.  Notice that the rules are exactly similar (although in different order).  Thirdly, the Find Laws algorithm generates a final exploration results report that appears in the Results panel.

Step 3: Understanding the Find Laws Exploration Report

This report is generated upon obtaining the first exploration result. The report is renewed each time a new significant result is obtained.  The report consists of two main sections, a text section and a graphical output section. 

About the Text Section and Rules Generation: The text section contains PolyAnalyst’s technical details of process time and process location, the best significant rule, the most exact rule, and other statistical characteristics of the obtained solution.  The best significant rule is the solution.  It is exactly equivalent to the formula used for data generation in the beginning of this tutorial!  In this dataset PolyAnalyst’s Find Laws algorithm explored the two variables and found the perfect relationship.

The second formula is called the most exact rule. The most exact rule is a rule that PolyAnalyst finds that gives better predictions of the data being tested but has not yet passed PolyAnalyst’s rigorous tests of statistical significance.  In this example it is the same as the first one, although in real cases it is often different. The more simple and reliable most significant rule is used in the majority of cases. The most exact rule should be used only in exceptional cases where small improvements in accuracy are vitally important. The report also furnishes the values of standard error, standard deviation, significance index, and R-squared for both formulae. These are statistical characteristics of the difference between the predicted values and actual data.

1.  Scroll down to the Graphical Output Section.

About the Graphs:  This section contains four different graphs: Gain Chart, Predicted Versus Real, Predicted and Real Versus Counter, and Residuals.

Gain Chart:  The gain chart is not covered in this tutorial as it does not apply to this dataset.  Gain charts are used in marketing analysis and will be discussed in depth in a later tutorial.

Predicted Versus Real: This graphs the predicted values of the target attribute calculated according to the most significant rule versus the real values of the target attribute.  The closer that points from both attributes lie to the diagonal, the better the quality of the prediction.  In this case the predicted y values lie exactly on the real values.  PolyAnalyst demonstrates its mathematical power by discovering the perfect equation of the line, paralleling the equation used to generate the dataset.

2.  Scroll down to view the real Versus Counter Graph.

Predicted and Real Versus Counter: This graph plots three attributes: the predicted value (target attribute value) and the real value on the Y axis, and the counter on the X axis, which is the record number in the world dataset (the row number in the imported CSV file).  In this case, only the red line is visible since the predicted rule matches the data exactly.  In other scenarios the predicted value, shown in blue, would be more recognizable when there is a difference between the predicted and real values.

3.  Scroll down to view the Residuals graph.

Residuals:  This graph facilitates visualization of the error distribution.  Each point on the residual line indicates what percentage of predicted points had a deviation from the actual point less than that indicated on the X-axis.  This graph allows one to visually evaluate the quality of the obtained model.

4.  Close the report window.

Congratulations. You have successfully completed the tutorial on simple and exact dependence, the first part of the PolyAnalyst Tutorials.
