Tutorial 3A: Exploration of Factors Influencing Gas Consumption by Cars

Brief description: In this lesson, PolyAnalyst is used to find an empirical rule that relates the MPG (miles per gallon) characteristic of a car, describing how many miles a car can run on one gallon of gasoline, with other technical parameters of the car.  Despite the fact that this example is too simple-minded to have an immediate commercial application, there exist many different ways to make use of the obtained results.  On one hand, the discovered empirical rule allows one to compare MPG of a given car with the value predicted by this rule.  For example, one can assess how economical a newly developed car should be and how that compares with the actual MPG of the created prototype.  On the other hand, the discovered rule provides one with an idea of what parameters are most important when one looks for the most economical car.  Such a rule could provide a rough guidance for designers of a new car who are constrained by certain limits on how economical the future car should be.  Finally, one can detect and summarize important relations and characteristics of different groups of cars.

 

Data Description: The data consists of 398 records characterizing various car types.  For each car type the following attributes are provided: the MPG value (mpg) measured for each car model in a test performed in 1982, the number of the engine cylinders (cyl), the cylinder displacement in cubic inches (displ), the engine power (power), the car weight in pounds (weight), a number of seconds required to accelerate to the speed of 100 miles per hour (accel), the car’s production year (year), the country of production (origin: USA, Europe, or Japan), and the name of the model (model).  You may load in PolyAnalyst and view the completed project file Autompg.prj, which contains all the results of the performed exploration in the PolyAnalyst project format.  However, we offer you the more useful alternative of using the provided raw data to develop your own data analysis project by following the step-by-step instructions that described below.  The data for the project is stored in the file Autompg.csv.

 

Step 1: Importing the Data

1.      Launch PolyAnalyst by double clicking on its icon.

2.      Select the Project | New... or alternatively you could choose the create project icon from the toolbar.

3.      The import wizard appears.  Select the Flat file radio button (Default option) and click Next.

4.      From the Open source window in the import wizard, select Browse.

5.      Select the Autompg.csv file in the file selection dialog box by double clicking on it.  Alternatively you can single click on the file and click OK. 

Default Location: C:\Program Files\Megaputer Intelligence\PolyAnalyst 4.1\Examples\Autompg.csv

6.      Click Finish.  You will see a dialog box for project editing (new project creation, in this case).

7.      Type the project name "MPG forecasting" in the Project Name line.

8.      Then change the type of some attributes, which by default are all treated as real numbers:

a.       The attribute cyl can have only integer values. Click on the Type column of the cyl attribute and choose Integer from the drop-down list that appears.

b.      Do the same for the attribute year by following the step above.

 

Note that PolyAnalyst has detected automatically that the origin and model attributes should be of the string type.

8.      Click OK.

Now the World dataset appears in the tree view of PolyAnalyst. It represents the master dataset, which contains all the data that has been loaded in PolyAnalyst.

9.      Select Project | Save As… from the pull-down menu.

10.  Specify the name “Mympg” in the file name text box.  Click Save.

PolyAnalyst adds the extension .prj to the filename automatically. It is important to specify where the project should be saved, so that the system can perform a periodic automated backup of the results of the exploration.

 

Step 2: Creating a Rule to Aid Exploration Engines

One additional step could be performed prior to starting the actual exploration: it is convenient to use the age of a car instead of the production year.  Since the survey was performed in 1982, the following formula could be used to calculate a car's age: age = 82 - year.  We use the age rule as apposed to using the year because the age of the vehicle is what matters, not the absolute value of the production year.

1.      Select Create Object | Create Rule to invoke the "Create a new rule" dialog box.

2.      Enter the name for a new derivative parameter in the Name text box.  Lets name this rule “age.”

3.      In the Rule Text field type "82 -"

4.      In the “double-click to insert field text box” double click on Attributes, then Year in the list at the right.

5.      In the comment field type "Age of the car at the moment of the test" and click OK.

An icon representing the typed rule appears under Rules in the tree view.  Now you need to apply the rule to the World dataset to calculate its values for all records.

6.        Right click on the age rule and from the pop-up menu select Apply to | World.  (Alternatively, you could drag the age rule onto the World dataset’s icon in the tree view by holding down the right mouse button over the rule and then releasing the right mouse button over the World dataset’s icon.)

7.      You can then open the World dataset by double clicking on its entry in the tree view to view the newly created attribute: the last column of the dataset table displays the value of this attribute.

 

For the future analysis an additional attribute “age1= 83 – Year” will be useful as well. This attribute does not have any zero values, which facilitates easier model building using the Find laws data exploration engine because of the decreased likely hood of dividing by zero.

8.      To create the “age1” perform the steps similar to those described in the above steps 1-5, using Create Object | New Rule and in the rule text field type “83- Year”.  You will be applying the age1 rule in step 4. 

 

Step 3: Exploring the Data Set

Next you are going to view the Summary Statistics for the World dataset.  This is usually one of the first steps to analyzing the data.

1.      Right click on the World entry in the tree view and choose Explore | Summary Statistics from the pop-up menu.

2.      Then click OK on the Summary statistics of dataset – world window.

About the SS_World Report: The vehicles have an average of 23.51 miles per gallon and an average of 5.455 cylinders.  The majority of the vehicles, in the world data set, are form the USA with a total of 249 vehicles.  The model attribute has 305 different values out of 398 total (the appropriate line has been highlighted in the figure below).  There are too many different values compared to the total number of records to provide any useful insight concerning the relations hidden in the data, and thus this attribute should be excluded from the further exploration.

 

Step 4: Create a New Dataset

Let us create a new dataset that contains only the attributes that we found to provide further insight for our exploration  – this dataset will be the one actually explored. 

1.      Right click on the World dataset and choose Create New… from the pop-up menu.  (Alternatively, you could choose Create Object | Create Dataset from the menu bar.)  Type the new dataset name, Explored, in the Name field.

2.      Right click to deselect the fields Year, age, and model in the new dataset.  The fields that will be in the Explored dataset are highlighted in blue.  Click OK.

Now the new dataset Explored has been created.  To complete the Explored dataset the rule age1 needs to be added.

3.      Right click on the age1 rule and from the pop-up menu select Apply to | Explored.  (Alternatively, you could Drag the age1 rule onto the Explored dataset’s icon in the tree view by holding down the right mouse button over the rule and then releasing the right mouse button over the Explored dataset’s icon.)

 

Step 5: Using and Understanding the Find Dependencies exploration engine

Prior to searching for an explicit form of the dependence of mpg on other attributes, we can perform one additional preliminary investigation.  We wish to carry it out because real world data rarely contains only "good" records, which are free of errors and obey some well-defined relation.  Actual data often contains errors, exceptional cases, or anomalies.  It would be important to single out such exceptions for two reasons: first, these exceptions hamper finding explicit relations between attributes in data.  If they are not too numerous, such exceptional records should be simply excluded from the data.  Second, these exceptional cases might be interesting on their own, because they might represent some important, dangerous or, on the contrary, highly attractive situations.  In order to determine possible exceptional cases in our data about cars, we launch the Find Dependencies exploration engine.

1.      Right click on the Explored dataset.

2.      From the pop-up menu select the Explore | Find Dependencies (FD).

3.      In the appearing dialog box double right click on the mpg attribute to make it the target.

4.      Select the Liberal algorithm modification in the Algorithm Type field.

The Liberal Find Dependencies algorithm is best suited for finding anomalies in data.

5.   Name the process “FD_Liberal” and click OK to launch the exploration.

In a few seconds the exploration is finished and the system provides a report to the user.

The final report consists of the text report, a table report, and four graphs – a Gain Chart, Predicted vs. Real, Predicted and Real vs. Counter, and Residuals. The text window contains a lot of information about the detected dependence of the attribute mpg on weight, accel, and age1 attributes.

We are currently interested in only one record of this report: N of points obeying the dependence = 396.  This means that out of 398 records in the Explored dataset 396 can be considered as "normal", while two records represent exceptional cases.  In the Predicted vs. Real graph, those points that correspond to normal cases are marked by red, while exceptional points are marked by blue.

Predicted vs. Real Graph

Let us analyze the prediction table in the text window to see what cars comprise this class.  The table has 2 x 2 x 2 dimensions. Each cell of the table contains the points for which the parameters weight, accel and age1 lie in a certain range.  For example, all the cells in the upper layer of the table contain the records corresponding to cars of less than seven years of age (age1<8).  Each cell of the table contains four numbers.  We are now primarily interested in the second (points (cells)) and fourth number (points (subpop)).  These are the total number of records corresponding to this cell, and the number of "good", non-exceptional records in this cell.  Correspondingly, we see that, in our example, only one cell contains the two exceptional cases.  This cell represents relatively new (age < 8 years), heavy (weight > 2807 lb.), and quickly accelerating (accel < 15.5 sec) cars.  Note: You’ll need to switch the Weight drop-down box to [2807+] to see this cell.

Prediction Table

When we close all windows, we can see that a new dataset FD_Liberal appears in the tree view.  The Find Dependencies exploration engine that we have utilized created this dataset automatically.  The dataset contains all non-exceptional records.

 

Step 6: Creating and Analyzing a Complement

In order to single out the exceptional records, create a complement to the FD_Liberal dataset.  A complement contains all the records that are found in the World dataset but not in the selected dataset.

1.      Right click the FD_Liberal dataset and select the Create Complement option on the dataset pop-up menu.

2.      Name the created dataset Exceptions and click OK.

Now lets find out what car models correspond to the exceptional records.

3.      Right click on the Exceptions dataset and from the pop-up menu select the Edit… item.

4.      Right click on the model and age attribute to include it in the Exceptions dataset and click OK.

5.      Open the Exceptions dataset by double clicking on its entry in the tree view

Let us look more carefully at the exceptional records.  Adjust the length of the columns with the mouse so that you can see the full names of the car models.  The two exceptional cars are the Datsun 280-zx and the Toyota Cressida.

The first observations that we can make are that both cars are made in Japan and are relatively new: two and one year old, correspondingly.  To be able to say more about their exceptional features we need to compare the Summary Statistics for these two cars with the rest of the data.

6.      Right click on the Exceptions dataset and choose Explore | Summary Statistics.  Click OK.

From the comparison with the statistics for the main body of the data we see that the two exceptional cars have a significantly better than average acceleration time: their values of the accel attribute are close to 12 seconds, compared to about 16 seconds on the average. Note: You can double-click on the SS_World report if you want to bring the overall summary statistics back on screen.

 

Step 7: Creating A Histogram

Thus, to understand why the Find Dependencies engine classified the two cars as exceptional we need to compare them to other cars, which accelerate quickly.  Let us look at the distribution of the values of acceleration over all data using a Histogram.

1.      Choose Create Object | Create Histogram from the menu bar.

2.      In the opening dialog box, name the histogram Acceleration.

3.      Click the World dataset to select it.

4.      Click the accel attribute to display it on the X-axis. Click OK.

For a better visualization of the data, right click on the histogram and select Scale from the pop-up menu.  Set MinX to 8, MaxX to 25, and the number of Intervals to 17, and click OK.  We see that the acceleration time peaks around 15 seconds.

About the Histogram: Along the X-axis are various ranges of the accel attribute.  Along the Y- axis are the number of values that fall into the range.  The graph clearly shows that the accel range from 14.0 to 15.0 has the most records, which is around 65.

We can separate the fast accelerating cars from the slow accelerating cars by setting a split at 13 seconds.

 

Step 8: Splitting and Editing a Dataset

Split is a function used to easily split a dataset according to criteria you set.

1.      Right click on the Explored data set and from the pop-up menu select Split | to equal intervals…

2.      Click on the accel attribute in the list of attributes in the appearing dialog box, enter 13 in the from field and 100 in the Step field, then click OK.

The records in this dataset correspond to the quickly accelerating cars.  You also may simply delete the second dataset, accel_2, which corresponds to slowly accelerating cars – we will not need it for further analysis.  To do this, either right click on the dataset in the tree view and click Delete, or simply select the dataset and press the Del key on the keyboard.

3.       Right click on the accel_1 dataset in the tree view and select edit.

a.       Rename it as “Quick” in the Name text box

b.      Include the age attribute instead of age1 in the dataset using the right click.

4.       Click OK.

 

Step 9:  Comparing Datasets using Summary Statistics

Now let us calculate the Summary Statistics for the Quick dataset and compare it with the Summary Statistics for the two exceptional records singled out by the FD engine.  This will allow us to see why the FD engine considered the two records as exceptional.

1.      Right click on the Quick dataset and choose Explore | Summary Statistics.

2.      Re-open the summary statistics for the exceptional records in the reports collection by double clicking on the SS_Exceptions icon.

Now we see that the two exceptional cars are extremely economical for their class: compare their average MPG of 29 to the average MPG for the quickly accelerating cars, which is less than 17.  We note that the exceptional cars are smaller on the average in the number of cylinders, cylinder displacement, power, and weight.  Lastly, we observe that these two cars are made in Japan, while the majority of quickly accelerating cars are made in the USA, and that the exceptional cars are much younger than others.  These cars have been produced in the beginning of the eighties, while other quickly accelerating cars were produced in the early seventies.

 

Step 10:  Using the Cluster Exploration Engine

The FC engine requires no target attribute to be specified, as it finds areas of the dataset that are similar in any respect.  As such, it is a form of undirected or exploratory data mining, often used as a starting point before moving on to other more targeted data mining techniques.

1.      Right click on the Explored dataset and select Explore | Cluster (FC).

2.      Select strict as the algorithm type. Click OK 

In a few seconds the FC engine terminates its work and reports that it has found two clusters in the data.

About the Report: The report lists the attributes that had been utilized for locating the clusters, displays the probability that the detected clusters occurred by chance (about 3 x 10-28 in the present example), the total number of points entering each of the clusters, and the number of detected clusters.  It also marks by different colors the cells of the clustering table containing points that belong to the different clusters. The rest of the data points, which have a more homogeneous distribution, are left uncolored in the clustering table.

The FC engine creates a number of new datasets, which correspond to the detected clusters: two new datasets in the present example. Rename the datasets that correspond to the two found clusters.

3.      The cluster named FC_Explored_1 (marked blue in the table) contains the most powerful and quickly accelerating cars: name this dataset High_Power by right clicking on the icon and selecting edit from the pop-up menu and changing the Name text box.

4.      The cluster named FC_Explored_2 (marked red in the table) contains the least powerful and slowly accelerating cars: name this dataset Low_Power by following the last step.

 

Step 11:  Comparing Data Sets Using the Snake Chart

A snake chart is used to quickly compare two or more datasets across multiple attributes.

In this step we will compare the Explored, High_Power, and Low_Power data sets.  The snake chart will compare the average of every attribute among the three data sets.

1.      To launch the snake chart select Create Object | Create Snake Chart from the main drop-down menu.

2.      In the Snake chart dialog box name the chart “Explored_vs_HP_vs_LP”.

3.      From the data sets box select the data sets Explored, High_Power, and Low_Power by right clicking on their names.

4.      We do not want to deselect any attributes from the attributes box because we want to compare all of them.

5.      Check the Normalize by Dispersion box.  This will allow us to see a quantitative difference between the values of attributes in the juxtaposed datasets more clearly.  Click OK.

 

 

About the Snake Chart:  The snake chart compares which datasets have the highest or lowest average of a given attribute.  When the Normalize by dispersion is checked the graph indicates exactly how much the datasets vary per attribute.  On the other hand, when Normalize by dispersion is not checked the graph indicates the data sets’ difference using 1’s (High) and 0’s (low). 

In the appearing graph you can see that the Low_Power data set (green) has the longest acceleration time, the High_Power data set (red) has the quickest acceleration time, and the explored data set (blue) is in-between the other data sets.  This type of explanation can be made for every attribute in the chart above.

 

Step 12:  Compare Two Clusters Using Summary Statistics.

1.      Right click on the High_Power dataset and choose Explore | Summary Statistics.

2.      Follow the same step above for the Low_Power dataset.

3.      Arranged Both the High_Power and Low_Power Summary statistics so they can be viewed simultaneously.

The main conclusions are that the most powerful and quickly accelerating cars were made predominantly in the USA in early seventies, have on average very low MPG (15), many cylinders (8), large cylinder displacement (350), and are heavy (about 4000 lb.)

 

The least powerful and slowly accelerating cars were made mainly in the Japan in late seventies, have on average very high MPG (32), few cylinders (4), small cylinder displacement (100), and are light (about 2000 lb.)

 

The obtained results confirm the well-known facts in the considered example.  However, there are might be other situations when one has no previous knowledge about the field.  Then, the Cluster engine can provide a very quick and direct way to learn important characteristics of the existing clusters in the data.

 

Step 13: Comparing Datasets Using a Real 3D Chart

Let us use a real 3D chart to visually see the difference between the High_Power and Low_Power data sets.

1.      To launch the real 3D chart select Create Object | Create Real 3D Chart from the main drop-down menu.

2.      Name the chart “High_vs_Low”.

3.      Now we need to select attributes to be on our axes.  For the X-attribute choose age1, Y-attribute choose weight, and for the Z-attribute choose mpg.

4.      Check the Fog Check box.

5.      Next select the data sets High_Power and Low_Power by right clicking on them.  Click OK.

About the Create new real 3D chart: Point size option allows you to change the size of the spheres.  The Fog option, when checked, will give the spheres in the background (furthest from you) a blurry shading so it is possible to clearly distinguish the depth of the spheres.  The Auto scale option, when checked, will automatically arrange the scale of the 3D diagram so all spheres will be viewable.  This option should always be checked when using the real 3D chart because the 3D cube cannot be scaled manually.

About the 3-D Graph: Locate the base corner marked by a “0”.  Each axis spreads out from this base point towards a predetermined value that serves as the dimension of the cube, such that length = height = width.  The graph may be rotated in PolyAnalyst to view the cube from different perspectives.  Coordinates (x,y,z) are represented as different colored small spheres. In this snapshot of the 3D Chart the attribute weight runs along the vertical axis (Y-axis).  What this means is that the higher a sphere appears in the cube in a vertical sense, the greater the weight attribute of that specific sphere/coordinate.  The mpg attribute runs along the Z- axis.  This means that the farther left the sphere is the greater the mpg the vehicle gets.  The age1 attribute runs along the Y-axis.  Meaning the closer the sphere is to you, the greater the age of the vehicle.

 

On the appearing graph you can clearly see high power cars, represented by the red spheres, weight more, get worse gas mileage, and are typically older than the low power cars, which are represented by the blue spheres.
