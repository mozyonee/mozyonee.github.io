Tutorial 4: Gold Price Forecasting Using Stock Analysis

Brief Description: PolyAnalyst can be used to aid in predicting the direction of the change of the price of gold (referred to as DCPG from here on) based on analysis of stock indices.  Predicting the DCPG is very valuable in developing a profitable trading strategy.  PolyAnalyst attempts to predict the DCPG by using machine learning engines to explore and analyze data.

The goal is to predict the DCPG between two sequential sessions of the COMEX trading session.  We are more interested in predicting the DCPG than actual values of change in the price of gold.  This is because direction is something we can successfully predict with a great degree of accuracy.  The actual values of the change in the price of gold are much more complicated to predict, and this type of prediction is not pursued in this tutorial.

About the Data: The data spans 1,077 trading sessions, a time period of about 3 years.  The data used for this tutorial consists of:

·        Various stock market indices values, such as the Standard & Poor’s 500

·        Indices characterizing the performance of individual industry groups

The data is stored in the file COMEXGLD.CSV.  Each record in the file corresponds to the values derived from one COMEX trading session (day).  The differences between today’s and yesterday’s stock market indices are listed sequentially in the file.  The attribute names are contained in the first row.  These names are abbreviations of the indices.  The last attribute in each record is the value of the gold price change (dGC).

The Problem: Given the difference in values of various stock indices from yesterday and today, attempt to predict the DCPG for the next day’s trading session.

Step 2: Using Rule Assistant to Manually Create a Rule

PolyAnalyst provides the ability to create rules or modify rules found by exploration engines.  Rules are expressions that use attributes to accomplish a mathematical and or logical process. Data mining still requires people to sometimes enter information to further exploration and analysis by the computer.  PolyAnalyst makes this process of entering information as painless as possible by using the Rule Assistant.  The Rule Assistant is an easy to use interface for creating or modifying rules.  The Rule Assistant provides buttons for common mathematical operators, and lists of attributes and other functions all within reach of a mouse click.

We want to predict the DCPG for the next day’s trading session.  To predict the DCPG we will first try and create a model from the data contained in the imported file.  To create the model we wish to concentrate specifically on the values of the dGC attribute for each record.  The dGC attribute is the amount of the change in the price of gold.  We must work backward by creating a model that looks at the next day’s dGC value and explores how next day’s dGC is related to the attributes in today’s record.  Once we understand that model, we can then use the values of today’s attributes to predict the DCPG.

PolyAnalyst works with cases record by record.  To compare next day’s dGC to today’s attributes, we must somehow move the next day record up one record, so that it is in the same record as today’s other attributes.  To move this record up, we must create a new attribute that is the value of the next day’s dGC in today’s record.  Then PolyAnalyst can explore the data correctly.  To create this next day’s dGC attribute we will create a rule.

1.      From the main file menu, select Create Object | Create Rule…  The Create a new rule window appears.  This window is also referred to as the Rule Assistant.

2.      Name the rule TomorrowGCDir.

3.      NOTE: The next section of creating the rule tests your ability to create a rule on your own, but the answer is located just underneath the section so scroll slowly!

We must somehow formulate the attribute next day’s gDC.  This may also be thought of as the next records gDC.  PolyAnalyst provides a basic expression that an attribute’s next row value can be stored in the original row.  The syntax is to add [1] after the attribute, signifying to take the value of the next row attribute.  So dGC[1] is the next row value of dGC.  What this really means is that dGC[1] is the value of the amount of the change in the price of gold for the next day.

We wish to create a conditional statement.  In programming languages this usually refers to an if statement.  A conditional statements evaluates a condition, and if the condition is true returns a true value, and if the condition is false returns a false value, and if the condition cannot be applied to something an otherwise value.  PolyAnalyst uses a form of the if statement to create a conditional expression.  In PolyAnalyst you can specify the true, false, or otherwise value that is returned after evaluating the condition.  This return value is the value of the TomorrowGCDir attribute for each record.

The condition is whether or not dGC[1] is positive or negative or zero.  If it is positive then we know that the DCPG is positive, so we return a positive value.  If dGC[1] is negative, then we know that DCPG is negative, and so we return a negative value.  If dGC[1] is neither positive nor negative, then there was no change between two records, and so we return a zero value.

The syntax of the functions used in PolyAnalyst may seem difficult at first but many functions are quite easy.  For a conditional if statement, the basic syntax is as follows: if(condition, value if true, value if false, otherwise).  PolyAnalyst provides the functionality to create nested if statements too.  In this case, testing for the negative value of dGC[1] requires a nested if statement.

4.      Try and enter the following statement into rule box on your own by translating this text rule into a real mathematical rule with correct syntax.  Note that you can use the Rule Assistants several GUI elements to make creating the rule very easy.

If the next day value if dGC is positive, then return a positive number (use one).  Or else, if the next day value of dGC is negative, return a negative number, or else return a number signifying no change.

The answer is shown following this picture.

The answer is:     if(dGC[1]>0,1,if(dGC[1]<0,-1,0))

5.  Click OK.  The rule is created and its icon appears under the rule section in the Workspace panel.

Step 3: Applying the Rule to a Dataset

PolyAnalyst provides the user with a functional interface supporting drag-and-drop techniques.  In this tutorial, applying the rule is as easy as drag-and-drop.

5.      Right click the TomorrowGCDir rule in the Workspace panel.

6.      Select Apply to | Dataset World.  PolyAnalyst applies the rule.

If you double-click on World, you’ll see that the last column in the World dataset contains the values of the new attribute TomorrowGCDir.

Step 4: Using Multiparametric Linear Regression

We have now successfully setup an attribute in each record that can be used as the target attribute in Linear Regression.  This will allow us to determine the influence of the other attributes in each record to the TomorrowGCDir attribute, as Linear Regression in PolyAnalyst explores a case record by record.

1.      Right click on the World data set. Select Explore | Linear Regression (LR).  The Linear Regression on Data set – World window appears.

2.      Name the report LR_TomorrowGCDir.

3.      We are trying to predict TomorrowGCDir.  Double click on the TomorrowGCDir attribute to make it the target of the exploration (the dependant attribute). It should be highlighted in red.  Click OK.  PolyAnalyst creates the Linear Regression report under the Reports section in the Workspace panel and the report is opened in the Results panel.

The Linear Exploration Report: This report is divided into two main sections.  This report has been discussed in earlier tutorials and should be easier to understand at this point.

The Text Section:  The important information in this window is the expression modeling the dependence of the attribute TomorrowGCDir on the other attributes.

When reviewing linear regression, the numbers preceding each attribute are called coefficients.  Coefficients help relate the dependant attribute to an independent attribute.  They reveal the direction of the influence.  For example, TomorrowGCDir increases as dGSM increases, but decreases as dSPU increases.

The Terms: table:  The last two columns F-Ratio and part sum of squares display the significance of the influence of the corresponding attribute and its contribution to the explanation of the variability of the predicted attribute.  In the present case the two most significant attributes are dCRB and dGSM, due to their large F-ratios.  dCRB stands for the Commodities Research Bureau, and is a market index calculated by the United States Consumption Investigation Bureau.  GSM stands for Gold & Silver Mining, and is an index characterizing the prices for the stock of companies related to gold and silver mining.  All the attributes in the terms table can be better visualized by looking at the terms graph farther down in the report.

The Terms Graph:

In fact, the performed analysis is not required for obtaining the prediction for the gold price change direction. To have a forecast for this direction one could simply apply the found rule to the data containing the information about the future trading sessions. Nevertheless, the analysis of the obtained relation is very useful for the understanding of the market dynamics, and can be most valuable itself.

Step 5: Using the Rule Assistant

What percentage of cases in the provided historic data provide a correct prediction for the gold price change using the earlier obtained linear rule?  The prediction is correct when the corresponding values of the attributes LR_TomorrowGCDir and TomorrowGCDir are either both positive, both negative, or both zero.  To check this condition we create a new rule as follows:

1.      From the main file menu, select Create Object | Create Rule…  The Create a new rule window appears.

2.      Enter the following expression into the rule box manually or by using the Rule Assistant:

(LR_TomorrowGCDir*TomorrowGCDir>0) or (LR_TomorrowGCDir=0 and TomorrowGCDir=0)

This rule returns either true or false.  If the predicted and actual attributes both change in the same direction, the rule returns true.  Otherwise the rule returns false.  PolyAnalyst uses the term yes to denote true and the term no to denote false.

3.      Name the rule Dir_OK.  Click OK.  The rule is added to the rules section in the Workspace panel.

4.      Apply the rule to the world data set using the drag and drop method.  Open the World data set.  Notice that the last column is now this new rule.

Step 6: Using the Summary Statistics Exploration Engine

We can test our prediction by using Summary Statistics Exploration.

1.      Single Click the World data set to select it.  From the toolbar, click the  button.  The Summary Statistics of Dataset – World window appears.  Since we want characteristics of all the values, do not select or deselect any values.  Click OK.  The Summary Statistics Report appears in the Reports section of the Workspace panel and opens in the Results panel.

Summary Statistics Report: The Statistics report is divided into a text section and a chart section.  The text section contains a table for integer attributes and a table for logical attributes.  Look at the logical attributes table Dir_OK attribute.  In 613 out of 899 cases the attribute Dir_OK has the value true.  In ratio, this means that in 68% percent of the cases, or records in the dataset, the Linear Regression Exploration Engine prediction for the direction of the change of the gold price is correct.

Of course, this preliminary result is of quite limited practical value. It is far more important to predict the amount of the price change in addition to its direction. However, this task is less trivial and requires the utilization of other exploration engines of PolyAnalyst. Nevertheless, we have just completed the first important step in building a successful trading strategy. If you would like to continue this exploration on your own, try using Find Laws or PolyNet Predictor rather than Linear Regression and examining the results. Both algorithms will give more accurate predictions than Linear Regression with this data.
