Tutorial 6: Direct Marketing Analysis

Brief Description: In this lesson, PolyAnalyst is used to predict buyers of a service based on easily available data about a set of businesses.  This tutorial is based on an actual data analysis done by Megaputer for a telecommunications company, though the numbers and data fields have been changed and the amount of data reduced and simplified to speed the analysis.  The client was introducing a new product, and would thus have to call prospective customers or send out mass-mailings to companies that would be interested in a new telecommunications service.  This is an expensive proposition, and the company turned to data mining to find a way to reduce the cost of direct marketing while increasing the payoff.

 

Direct-marketing companies have to spend money for every possible customer they contact, whether it’s by telephone, mail, fax, or in person.  The majority of the customers contacted never respond to the offer; the percentage of people contacted that do respond is called the response rate.  If the response rate is too low, a company may find itself spending more money on contacting customers than it makes in sales to the customers contacted.  If there were some way to improve the response rate by predicting which customers were going to respond before spending the money to contact them, it would greatly improve the profitability of any direct-marketing company.

 

Data mining with PolyAnalyst provides this predictive power.  The data in this case consists of publicly available business data on customers contacted in the past, along with a flag variable indicating whether or not they responded to the offer.  From this data, PolyAnalyst will be used to generate a predictive model of offer response in the future based on business data.  The direct-marketer can then reduce their marketing expenses by only contacting the prospective customers that PolyAnalyst predicts will respond, rather than contacting all prospective customers or a random set of them.

 

The Data: The data consists of 13,117 records characterizing different prospective customers, only some of which responded to the offer.  For each prospect, the following attributes are provided: the number of employees both at a particular office (Local_Emp) and for the entire company (Ttl_Emp), the annual sales in thousands of dollars at a particular office (Local_Sale) and for the entire company (Ttl_Sale), whether or not the company does business outside the United States (International_Flag), a code indicating the amount of spending the company does each year on advertising (Ad_Spending), whether the company is operating from the same location as before, has moved recently, or is a new business (New_Location), whether the company is privately or publicly owned (Owner), an industry type code, both specific (Pri_Ind_Type) and general category (Pri_Ind_Cat), and the age of the company in years (Age).  None of these has an obvious connection to whether or not the company is likely to purchase a new telecommunications service – but PolyAnalyst can be used to find these connections. The data for the project is stored in dmarket.csv.

 

Step 1: Importing the Data

1.      Launch PolyAnalyst by double clicking on its icon.

2.      Select the Project | New...  from the drop- down menu or alternatively you could choose the create project icon from the toolbar.

3.      The import wizard appears.  Select the Flat file radio button (Default option) and click Next.

4.      From the Open source window in the import wizard, select Browse.

5.      Select the Dmarket.csv file in the file selection dialog box by double clicking on it.  Alternatively you can single click on the file and click OK. 

Default Location: C:\Program Files\Megaputer Intelligence\PolyAnalyst 4.1\Examples\Dmarket.csv

6.      Click Finish.  You will see a dialog box for project editing (new project creation, in this case).

7.      Type the project name "Direct Marketing" in the Project Name line.

8.      Then change the type of some attributes, which by default are all treated as real numbers:

a.       The attribute Pri_Ind_Type (industry type) is predetermined as numerical, though these numbers are really different categories of industries.  Click on the Type column of the Pri_Ind_Type attribute and choose Category from the drop-down list that appears.

b.      In addition, change the types of Age, Local_Emp, Ttl_Emp, Local_Sale, and Ttl_Sale to integer, as the data in these columns contains only whole numbers.

c.       Set the attributes Buyer_Num and Buyer_Cat to yes/no.

9.      The remaining types can be left alone, as PolyAnalyst has already identified the types of those attributes.

10.  Finally, uncheck the check box next to the Random_Number field – we have no need for the field in this exploration. Click OK to create the project.

 

Step 2: Preprocessing to make date more meaningful

 It is possible to work with the data in its current form, but sometimes a few transformations on the data beforehand can make for more meaningful results.  Just as in Lesson 3, when we converted a Year field into an Age rule to get a clearer relationship with the other attributes, we can create rules based on this data to improve this analysis.

 

Local sales (Local_Sale), total sales (Ttl_Sale), local employees (Local_Emp), and total employees (Ttl_Emp) vary wildly based on the size of the company.  All four of them can be used as measures of company size – though they also are influenced by other important characteristics (the productivity of the company, the number of branch offices, the profit margin, etc.), these factors are overwhelmed by the effect of company size.  For instance, a huge company with a very low profit margin and unproductive employees will still have higher values in all these fields than a smaller company with a high profit margin and very productive employees.  A better set of data would have a single measure of company size, and use ratios to indicate these other characteristics.  The ratios will help bring out the other characteristics of the companies by removing company size from consideration.

 

These ratios can be created in PolyAnalyst as rules.

1.      Choose Create Rule from the Create Object menu, and name the first new rule Emp_Ratio. This ratio will be that of local employees to total employees.

2.      Open the Attributes tree on the right-hand side of the Create Rule dialog box, and double click Local_Emp.

3.      Then click the / key at the top of the window to indicate division, and choose Ttl_Emp from the Attributes tree.

4.      Click OK to create the new rule.

5.      Repeat this process for a rule Sale_Ratio, defined as Local_Sale / Ttl_Sale, and Emp_Sale, defined as Local_Sale / Local_Emp.

                                                                                                                                                                                                     

Explanation of Ratios: By using the Local_Emp attribute as a measure of company size and using the three newly created ratios instead of the other three attributes used in creating them, thus eliminates problems of using the attributes. The ratios provide a meaningful comparison of the characteristics of companies even if the companies are of vastly different size, while the Local_Emp attribute provides a measure of company size so no data is lost.  The ratio of local employees to total employees (Emp_Ratio) illustrates the size of the branch office compared to the overall company – this will be lower for companies with smaller, more spread-out offices.  The ratio of local sales to local employees (Emp_Sale) illustrates employee productivity by showing how many dollars in sales are generated per employee.

6.      The last step in creating these rules is to apply them to the World dataset. Open the Datasets tree and the Rules tree in the main PolyAnalyst window.

7.      Right click on the Emp_Ratio rule and select Apply to| World.  Equivalently, you can Click and drag the Emp_Ratio rule from the Rules tree onto the World dataset.  Repeat this process for the Sale_Ratio and Emp_Sale rules. This adds the new rules to the dataset.  Alternatively, you can check all three Rules in their check box and drag and drop them all at once by selecting one of the three.

Tree View

 

Step 3: Creating an Exploration Dataset

The next step is to create a dataset to work with.  The World dataset contains attributes we don’t want to consider in our analysis at this time.

1.      Make a new dataset by selecting Create Dataset from the Create Object menu. 

2.      Name the dataset Explored in the Name text box.

3.      Deselect the attributes: Ttl_Emp, Local_Sale, or Ttl_Sale.  These attributes are no longer needed due to the ratios we created.

Step 4: Beginning the Analysis

We will begin the analysis with a look at the summary statistics for the new Explored dataset.

1.      Right click on the Explored dataset, choose Explore, then choose the Summary Statistics engine and click OK.

About the Summary Statistics Report:  First of all, under Logical Attributes, the dataset contains 44% buyers and 55% non-buyers – obviously, in the real world a response rate of 44% would be ridiculously high.  This is because of the way the data in this dataset was chosen – the company selected a list of known responders, and then added to the list a few thousand random companies (not contacted in the past) to serve as example non-responders.

 

Second, the Pri_Ind_Type attribute (shown under Categorical attributes) takes on 78 different values, and the number of companies in each category varies widely – from a low of 3 to a high of 1380.  This attribute is probably not a good one to include in our analysis – a few coincidental similarities in companies found in the low-frequency group could produce rules that aren’t really accurate.

2.      To remove this attribute from further consideration, choose the Explored dataset from the tree, right-click on it, and choose Edit.

3.      From the Edit Dialog box deselect the Pri_Ind_Type attribute, and click OK.

 

Step 5: Undirected Data Mining

At this time, we are ready to begin machine-learning analysis, the true power of PolyAnalyst.  We’ll start with an undirected analysis using the Cluster engine, to get an idea of what patterns in the data may be significant.

1.      Right-click on the Explored dataset and choose Explore | Cluster.

2.      Deselect the Buyer_Cat and Buyer_Num attributes by right clicking on them – our intention is to find patterns in the data we’re not already aware of, and it will do us no good if the system simply reports to us that there is a cluster of buyers and a cluster of non-buyers.

3.      In addition, exclude Local_Emp from the exploration by right clicking on it, so we don’t simply get a cluster of small companies and a cluster of large ones.

 

 

4.      Execute the exploration by clicking OK.

About the Report: The Report gives the most influential parameters as the attribute Age and Emp_Sales.  The clustering algorithm creates eight new datasets, which are FC_Explored_1 through FC_Explored_8.  Though we removed the Buyer_Cat attribute to do the clustering, now that the dataset has been split it is one of the attributes we’re most interested in.

5.      Right-click on FC_Explored_1 and choose Edit…

6.      Turning the Buyer_Cat attribute back on by right clicking on it.  Click OK.

7.      Repeat this for the other seven datasets produced by clustering.

Now we’ll compare the summary statistics of these eight datasets.

8.      Right-click on FC_Explored_1, choose Explore | Summary Statistics, and click OK.

9.      Repeat this for the other seven datasets created by clustering, and examine the reports.

Look at the frequencies chart for FC_Explored_1, found at the bottom of the Summary Statistics report.

10.  Select the Buyer_Cat attribute to view.  Change the graph to a Pie or Doughnut type for easier visualization.

Analyzing Results of SS reports: Notice that it contains only 44% buyers – less than the World dataset.  Compare it with the other seven clusters – the second cluster is 16% buyers, also not a good market.  However, two of the clusters are a different story – cluster 4 contains 76% buyers, while cluster 6 contains 91% and cluster 7 contains 99% -- all well over the ratio in the Explored dataset.  Here may be a part of our target market.

 

Reopen the FC_Explored report. Look at the cluster table, which is organized with Emp_Sales down the left side and Age across the top. Clusters 4 and 6 are at the left side, while the less profitable clusters are further to the right.  A conclusion that could be drawn from this is that younger companies are better customers for our new product – the older a company is, the less likely it will become a buyer.  Though Sales per Employee was also used in finding clusters, it doesn’t appear to be as important to our analysis – the extremely valuable Cluster 7 contains only 113 points, while Clusters 4 and 6 is much larger.  Though Cluster 7 shows a lower Sales per Employee, it actually shows that the Sales per Employee was only about average, while Clusters 4 and 6 showed it to be much higher.  There isn’t sufficient information here to decide if a low or high Sale per Employee is something to look out for.

 

Now we are going to analyze the eight clusters using a snake diagram.  The Snake diagram is an excellent tool for quick comparisons of two or more data sets.

11.  To launch the snake chart select Create Object | Create Snake Chart from the main drop-down menu.

12.  In the Snake chart dialog box name the chart “Cluster_Comparison”.

13.  From the data sets box select the data sets FC_Explored_1 to FC_Explored_8 by right clicking on their names.

14.  We do not want to deselect any attributes from the attributes box because we want to compare all of them.

15.  Check the Normalize by Dispersion box.  This will allow us to see a quantitative difference between the values of attributes in the juxtaposed datasets more clearly.  Click OK.

 

Snake Chart

About the Chart: The Snake Chart allows you to easily compare which data set(s) have the highest and lowest average attribute value(s).  Also, the snake chart gives you a brief visual overview of the Summary Statistics of the eight clusters.

 

At this point, we have one good piece of information – younger companies are more likely to be good customers – but no predictive rules and a few unanswered questions. At this time, we need to move on to directed data mining.

Step 6: Directed Data Mining

We can begin with a few rounds of the Find Dependencies algorithm, to determine what attributes have the greatest influence on the response rate.  Though we know from Clustering that Age is significant, we want to find other important attributes that Clustering missed.

1.      Open the Explored dataset, right-click on it, and choose Explore | Find Dependencies. 

2.      Name the process FD_1 (for Find Dependencies, 1st round), and double-click on Buyer_Num to select it as the target attribute.

3.      Right click once on Buyer_Cat to deselect it. Buyer_Num and Buyer_Cat contain the same data, but one is a numerical data type while the other is a categorical data type. Different exploration engines in PolyAnalyst require different kinds of target attributes, so both were included in the analysis – Find Dependencies requires a numerical target attribute. Click OK to begin the exploration.

Report on FD_1:  As Find Dependencies runs, it displays reports showing its findings so far. After a few minutes, the algorithm finishes and shows Age as the most influencing parameter. This is good to know, but to run a thorough analysis we need more than one attribute.

4.      Select the Explored dataset and run Find Dependencies again.

5.      This time, name it FD_2, set Buyer_Num as the target attribute, and deselect both Buyer_Cat and Age, so as to find the most influencing parameters besides age.

Report on FD_2:  The second round of Find Dependencies shows that Emp_Ratio and Local_Emp are also important to the exploration. This gives us three attributes to work with in future explorations.  It would be possible to run Find Dependencies even more times, but to do so too many times may indicate to us that every attribute is important in some way – since the goal was to reduce the number of attributes, we will stop here with three.

 

Find Dependencies has produced a variety of attributes found to be important. However, one of the greatest advantages of PolyAnalyst is the availability of a variety of exploration engines.  To check for other dependencies, we will run an additional round of preprocessing with the Linear Regression engine. Since the goal of our analysis is to classify people into groups (those who will respond to a direct-marketing offer and those who won’t), we will do this through the Classify algorithm. The Linear Regression will give us a meaningful classification rule, as well as giving us a list of influencing parameters.

 

Since this is only a tutorial, we will speed up the analysis by reducing the size of the dataset.

6.      Right-click on the Explored dataset and choose Sampling | Random.

7.      Name the new dataset Sample.

8.      Choose to sample 25% of the dataset. Click OK to create the sampled dataset.

Note: Your results from now on can deviate somewhat from the ones presented below, because random selection of data from the original data set is probabilistic in nature.

9.      Right click on the Sample dataset, and choose Explore | Classify.

10.  On the Classify menu, name the exploration CL_LR.

11.  Select Linear Regression as the process type.

Unlike Find Dependencies, Classify requires a categorical attribute for its target.

12.  Choose Buyer_Cat as the target attribute while deselecting Buyer_Num. Due to the way Linear Regression works, no intermediate reports are generated while the regression runs – a report is only produced once the exploration finishes.

 

 

About the Report:  The report produced after a few minutes showing a reasonably accurate prediction rule.  The report shows that the overall classification probability is 73.45% -- a high accuracy, considering that this is just a linear regression used for preprocessing. In addition, both the Class 0 and Class 1 errors are low – around 17% for Class 0 and 37% for Class 1.  These are useful parameters because sometimes a classification will achieve a high overall classification probability despite having misclassified one of the two classes severely.  Looking at the list of terms, we can see that Ad_Spending, Age, International_Flag, Local_Emp, New_Location, Owner, and Pri_Ind_Cat were all influencing terms.

 

In addition to this information, PolyAnalyst can generate Lift and Gain charts to illustrate the effectiveness of direct marketing campaigns. The Lift chart, shown on the report, shows the effectiveness of a targeted campaign.  Plotted on the X-axis is what percentage of all prospects have been contacted; on the Y-axis is what percentage of people who would respond were actually contacted.  The central diagonal indicates the use of random mailing– if you contact 50% of all prospects, you will likely contact 50% of the responders.  The curve above this axis shows the effect of targeting only those people that PolyAnalyst indicated were most likely to buy – in this example, contacting the top 50% of PolyAnalyst’s prospects results in reaching 70% of all the responders.

 

Lift Chart

 

The Gain chart provides an even more striking depiction of the effectiveness of the targeted campaign.  If the Gain chart does not appear click on the name “Gain chart” and the graph will appear.  The three boxes above the Gain chart are for cost per contact, profit per response, and maximum number of contacts. If, for instance, it costs $4 in phone expense and employee time to call a person and offer them a product, each person who is interested in the product generates an average of $10 of revenue, and you have enough employees to call 1,000 people per month, you would fill in 4, 10, and 1000 in the three boxes.  Fill in these numbers, and compare the two lines plotted on the graph.  The red line indicates your profit based on number of people contacted using a random strategy – the maximum profit of about $400 is achieved only by contacting all 1000 people.  The blue line, on the other hand, indicates profit based on the number of people called if you call those people PolyAnalyst has determined are the best prospects.  Profit is maximized at $1,372 – about three times as high – and by contacting only the best 600 or so prospects.  This is the effect of data mining being applied for direct marketing.  By contacting only the best prospects instead of all prospects at random, the marketer can achieve a higher profit than before by contacting less sales prospects.

 

Gain Chart

 

Step 7:  Advanced Analysis

Now that Linear Regression and Find Dependencies have found the most influencing attributes, these can be run on Find Laws, PolyNet Predictor, and Nearest Neighbor engines.  Look at the Partial Sum of Squares column on the Linear Regression report – notice that all of these are very high, except for the Owner entries, which are much lower. This indicates that except for Owner, all of the attributes used in the linear regression are significant.

1.      Click on the Sample dataset and select Explore | Classify. This time, name the process CL_FL and choose "Find Laws" as the process type.

2.      Choose Buyer_Cat as the target, and exclude from the analysis Buyer_Num, Owner, and Sale_Ratio.

3.      Set the time limit to 5 minutes; click OK, and the Classify engine will begin, giving intermediate reports as it progresses. After 5 minutes, a good classification rule has emerged.

 

 

About the Report: The rule has a 77% classification probability – notably higher than that produced by Linear Regression.  In addition, the Class 0 error (the rate of classifying buyers as non-buyers and thus missing prospective customers) is very low – 21%.  In a real analysis, this would be allowed to continue running for several hours, developing better and better classification rules and improving this accuracy even further, eventually finding a classification probability greater than 77%. In this case, the Find Laws output gave the best classification rule of any of PolyAnalyst’s exploration engines.

 

Interpreting the Results: Now that we have this classification rule, we can use the Classify report to analyze and interpret it.  The first thing to look at is the text report.  An overall Classification Probability indicates the chance for any given record that the rule will classify it accurately.  However, since in this case we’re doing market research, we would be much more concerned about accidentally classifying a buyer as a non-buyer, thereby missing a customer, than with accidentally classifying a non-buyer as a buyer and calling up someone who isn’t likely to buy the product.

 

The first line of the report states that if the prediction rule (see below) is greater than 0.4518 then value of Buyer_Cat is TRUE (class 1) otherwise it is FALSE (class 0).  This tells us what the classes are – in this case, we want to minimize the number of people accidentally classified into Class 0 (non-buyers).  Looking below, we can see that the Class 0 Error is only 21% -- an unusually good result for real data.

 

The charts below can give a more visual and business-oriented view of the value of this data.  These work the same way as the Linear Regression lift and gain charts; go ahead and enter 4, 10, and 1000 into the Gain Chart.  Note that this time profit is maximized at $1,542 – 13% higher than the $1,354 produced by Linear Regression!
