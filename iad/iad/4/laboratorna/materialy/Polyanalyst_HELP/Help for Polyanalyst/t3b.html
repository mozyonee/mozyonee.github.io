Tutorial 3B: Predictive Data Analysis on Factors Influencing Gas Consumption by Car

Lesson 3b is a continuation of lesson 3.  We recommend that you complete the lesson 3 prior to working on this tutorial. For your convenience we repeat the description of the data and the statement of the problem here. If you remember the attempted task from doing the lesson 3, you can go directly to Step 1 of the exploration below.

 

Brief description: In this lesson, PolyAnalyst is used to find an empirical rule that relates the MPG (miles per gallon) characteristic of a car, describing how many miles a car can run on one gallon of gasoline, with other technical parameters of the car.  Despite the fact that this example is too simple-minded to have an immediate commercial application, there exist many different ways to make use of the obtained results.  On one hand, the discovered empirical rule allows one to compare MPG of a given car with the value predicted by this rule.  For example, one can assess how economical a newly developed car should be and how that compares with the actual MPG of the created prototype.  On the other hand, the discovered rule provides one with an idea of what parameters are most important when one looks for the most economical car.  Such a rule could provide a rough guidance for designers of a new car who are constrained by certain limits on how economical the future car should be.  Finally, one can detect and summarize important relations and characteristics of different groups of cars.

 

Data Description: The data consists of 398 records characterizing various car types.  For each car type the following attributes are provided: the MPG value (mpg) measured for each car model in a test performed in 1982, the number of the engine cylinders (cyl), the cylinder displacement in cubic inches (displ), the engine power (power), the car weight in pounds (weight), a number of seconds required to accelerate to the speed of 100 miles per hour (accel), the car’s production year (year), the country of production (origin: USA, Europe, or Japan), and the name of the model (model).  You may load in PolyAnalyst and view the completed project file Autompg.prj, which contains all the results of the performed exploration in the PolyAnalyst project format.  However, we offer you the more useful alternative of using the provided raw data to develop your own data analysis project by following the step-by-step instructions that described below.  The data for the project is stored in the file Autompg.csv.

 

 

Step 1:  Data Mining with the Stepwise Linear Regression Engine

Let us try and find a model predicting mpg from other independent attributes. PolyAnalyst offers several exploration engines capable of addressing the task of predicting values of a numerical attribute: Find Laws (FL), PolyNet Predictor (PN), Memory Based Reasoning (MB), or Linear Regression (LR). First, we’ll use the most simple and fast Stepwise Linear Regression (LR) engine to see how well a linear model can predict the value of mpg.

1.      To launch the LR engine select the Explore | Linear Regression (LR) item from the pop-up menu of the FD_Liberal dataset (the two exceptional records have been excluded from this dataset – see Lesson 3).

2.      Double click on the mpg attribute in the opening dialog box to make it the target attribute.

About the LR Dialog Box: The dialog box allows the user to set a critical lower value for F-ratio (the ratio of a term's coefficient to the standard error of the determination of this coefficient), below which the corresponding regression term is discarded, and a maximal percentage of values of the target attribute that can be missing.  By default the critical F-ratio is set to 1.  In this example we would like to set the critical F-ratio value to 5, so that only the most important terms will be included in the model. 

3.      In the F-Ratio text box type in 5.

Leave the default maximal percentage of missing values - 50% - and click OK.

About the Report: First of all, the report displays a linear formula approximating the sought dependence.  The negative signs in the formula show that as the included independent attributes increase, mpg decreases.  Another important observation in the formula is that the USA was not producing economical cars in the 70s, which is shown mathematically by the –2.16287*1 (if the vehicle’s origin is USA).  The found relation has a rather high accuracy (R-squared is about 0.82) and high significance index – about 194 (as a rule of thumb, relations with a significance index of 3 or larger can already be considered as non-accidental), which means that this relation can be used for reliable predicting.  We observe that only a subset of all independent attributes are involved in the model. The included attributes are weight, age1, and origin of the car.  This is because the PolyAnalyst Linear Regression engine solves not just a single regression problem but also performs selection and stepwise addition to the model of the most important independent attributes until this process provides no further improvement of the results.  Thus, only most influential and significant independent attributes are included in the model, rather than blindly calculating the regression coefficients for all existing attributes.

 

Let us check whether the obtained linear model is supported by the common sense arguments.  We observe that the larger the weight or age of the car, the less economical the car is, as we could have anticipated.  The country of origin has a less significant impact (as can be seen in the Terms graph in the LR report), but we see that a car has lower mpg if it was produced in the USA - another well-known fact about American cars made in the seventies.

 

The lower left graph displays the predicted values of mpg, calculated according to the found linear formula, plotted against the actual values of the target attribute.

Predicted vs. Real

 About the Predicted vs. Real Graph: We can see that points on this graph are not distributed evenly around the main diagonal, which corresponds to an ideal prediction.  Predictions made in accordance with the linear formula are too low for the lowest and highest values of actual mpg, while they are too high in the middle.  This is an indication that a linear model works poorly for these values of mpg and that more powerful, nonlinear methods should be used, as is described below in detail.  The Terms graph below shows the relative importance of the independent terms entering the linear model in explaining a change in mpg.  We see that the main parameter influencing mpg is by far the weight of the car.

Terms Graph

About the Terms Graph: Out of the three involved independent attributes – weight, origin, and age1 – the weight and age1 attributes influence the target attribute mpg most significantly.  We also see that F-ratios for both terms entering the dependence are high, corroborating the importance of these terms.

 

Step 2: Data Mining with the Help of Visualization Tools

Let us visualize the role of the two attributes (weight and age1), which have been selected by the LR engine as the most influential with the help of a 3D-chart.

1.      Select Create Object | Create 3D-chart from the main menu.

2.      Name the graph "3D_MPG".

3.      Select weight to be depicted on the X-axis and age1 on the Z-axis (which is represented by the changing color).

4.      Press the Add Data Graph button, select the Explored dataset and then the mpg attribute by left clicking on their names, and click OK. After that select a marker for representing the data points on the graph and click OK.

 

 

3D-Chart

 

About the 3D chart:  On the appearing graph you can see that heavier cars are less economical: their mpg is lower.  An interesting observation is that this dependence is obviously nonlinear: the dependence of mpg on weight is more pronounced, and the curve is steeper, for light cars, while the relation gets weaker, and the curve flattens up, for heavier cars.  We also note that older cars have lower mpg, while for newer cars the dependence of mpg on weight is looser.  This graph suggests that an actual dependence of mpg on independent parameters is not linear, and thus the LR engine cannot do a good job in predicting the value of mpg.  Luckily, PolyAnalyst has other far more general and powerful exploration engines, PolyNet Predictor (PN) and Find Laws (FL), which are suited well for modeling nonlinear data.

Step 3:  Data mining with the PolyNet Predictor

Next, let us use the PolyNet Predictor exploration engine to predict the attribute mpg.  The PolyNet Predictor uses a neural network to predict the value of the target attribute based on the values of several other attributes. 

1.      Select the Explore | PolyNet Predictor…(PN) item from the FD_Liberal dataset pop-up menu (the two exceptional records have been excluded from this dataset).

2.      In the list of all attributes in the opening dialog box, double click on mpg to make it the target attribute.

3.      Leave the other options as default. 

 

 

4.      Click OK

About the Report: In this report you can see that R-squared is 0.837 which is better than the Stepwise Linear Regression model (where R-squared equals .8202) produced in step 1.  

 

Predicted vs. Real

About the Predicted vs. Real Graph:  As you can see the majority of the data points follow the main diagonal.  This shows us that the PolyNet predictor did well at predicting the target attribute.

 

Residuals Graph

The PolyNet Predictor is very beneficial when dealing with a large number of records where the Find Laws engine, discussed next, does not perform well when dealing with large amounts of data.  Also, PolyNet Predictor is good for predicting values of continuous variables, but is not a good tool for explaining the underlined relationships of the attributes.

 

Step 4: Data mining with the Find Laws engine

Next, let us use the Find Laws (FL) exploration engine to find an explicit form of the dependence of mpg on other parameters. The FL engine not only allows predicting the values of the target attribute, as the PN engine does, but also helps explaining how individual independent attributes influence the target attribute.

1.      Select the Explore | Find Laws (FL) item from the FD_Liberal dataset pop-up menu (the two exceptional records have been excluded from this dataset).

2.      In the list of all attributes in the opening dialog box, double click on mpg to make it the target attribute.

3.      Leave 0 as the default value for the desired standard error – that way, the process will work until terminated externally. We can set the time limit to 5 minutes – this forces the FL engine to stop automatically when this time elapses. Click OK.

You will see a live picture of the evolving model for predicting mpg. Each time a more accurate and significant model is obtained, the FL engine furnishes a report to the user. The displayed windows are the text report of the process, the Gain Chart, the Predicted vs. Real graph, the Predicted and Real vs. Counter graph, and the Residuals graph.

Predicted vs. Real

Residuals Graph

 

In five minutes the FL engine stops and we can analyze the final results obtained by the engine. The best significant rule is most reliable model for mpg obtained by the system.  This is the model that should be used for predicting mpg in future situations in the majority of cases. Note that PolyAnalyst automatically found this rather involved nonlinear relation that expresses the dependence of mpg on other attributes without any close supervision from the user.

 

 

The best exact rule is still more accurate in explaining the data, but it had not passed all the tests for statistical significance and may be over fitting the data or contain some insignificant terms.  The accuracy and statistical significance for each of the two models are provided below in the same window.  The significance of both models is very high (>100). You can observe that the accuracy of the model developed by the FL engine is much higher (R-squared = 89.08%) than the accuracy of the linear model (R-squared = 82%). Upon closing the report window, you see new entries for the results of the Find Laws exploration in the Rules and Reports containers in the tree view.

 

Don’t be alarmed if Find Laws did not come to exactly the same formula in 5 minutes as was illustrated in the tutorial screen shots.  Due to varying computer speeds, it may take more or less time to arrive at a given formula on different computers.  Your computer may have found a more or less accurate rule in the same time period.  Given enough time, Find Laws would arrive at the same conclusions on any computer, and would actually come to a better rule than the one illustrated here.

Step 5: Analyze Found Relations Graphically

In principle, we can analyze the developed relation to make conclusions about the combined influence of independent attributes on the target attribute.  However, this analysis can get quite complex. PolyAnalyst provides a much better and simpler way to analyze the found relation. The user has means to represent this multidimensional nonlinear dependence graphically.

 

Let us plot the predictions of both models, LR and FL, against the data on a two-dimensional graph.

1.      Open the dialog box for a 2D-chart creation by choosing Create Object | 2D Chart from the main drop-down menu.

2.      Name the graph MPG.

3.      You can change the background color of the graph by pressing the Back Color button.

4.      Select the X-axis value from the X Attribute list – we’ll use the weight of the car here.

5.      Then click the Add Data Graph button. Select the World dataset, and the corresponding attribute mpg by clicking on their names, then click OK.

6.      Click the Add Rule Graph button. Click on the names of the LR_mpg and FL_mpg rules in the list and click OK.

7.       Select the desired line and marker shape and the color for each selected graph in turn from a palette on the right. Then click OK.

 

 

 

 

About the 2D Chart:  The graph of the investigated relation is displayed, with the weight attribute shown on the X-axis, and all other independent attributes entering the models represented by sliders.  By moving a slider bar we can change the values of the corresponding attribute.  We have made use of the only possible way to visualize the found relation, since there are no means to represent it as a surface in four-dimensional space.  You can observe that the FL engine has done a much better job than the LR engine in modeling the mpg data for different cars.  While it may not seem that the Find Laws curved line fits the pattern of dots very well, remember that the high-weight cars tend to be old and high in power, while the low-weight cars tend to be new and low in power.  If you set Age and Power to their maximums, the curved line fits the data very well toward the right side – the high-weight side.  On the other hand, if you set Age to a low value (such as 2) and Power to a low value (such as 75), you’ll see that the Find Laws graph fits the left side of the graph very well.  Unlike the Find Laws graph, the Linear Regression graph just moves vertically as parameters are changed, and thus never gets as close to the outlying points at the edges of the graph.

 

Step 6:  Using and Understanding Decision Trees

Now suppose you have the problem of waning to predict the origin based on all parameters.  The target attribute is now categorical and we need to use an exploration engine that can handle this type of data.  Decision trees engine is a very robust tool for addressing classification tasks using categorical or Boolean data types.  We will use the decision tree engine to predict the origin of a vehicle using the FD_liberal data set. 

1.      Open the Decision Trees dialog box by right clicking on the FD_liberal data set icon and then choose Explore | Decision tree…(DT).

2.      Name the Decision Tree “Origin_Predictor” in the Process Name text box.

3.      Double click on the origin to make it the target attribute.

 

 

About the Decision Tree Dialog Box:  The target attribute of the decision tree must be a categorical or Boolean (Yes/No) value due to the decision trees purpose of classifying cases into categories.  The Maximum tree depth is the maximum number of branching points encountered when following any branch of the developed tree.  Minimum allowed node support is the minimum percentage of data records assigned to any node of the tree.

3.      Click OK.

About the Report:  The rule has an 85% classification probability.   The real / predicted table shows the result of running the data through the DT engine.  USA was correctly predicted 243 times out of 249.  Japan was correctly predicted 60 times out of 77, and Europe was correctly predicted 32 times out of 70.  The “Undefined” column contains records having missing values in attributes used in the Decision tree rule.

 

Note: Probabilistic character of Information Gain node splitting criteria of Decision Trees used in PolyAnalyst results in different (but close) classification results if more than one independent attribute provides a close information gain.

 

About the Tree Diagram:  The Diagram starts off with an initial split that produces two nodes (displ >= 134.5, displ <134.5 in our case).  The tree continues to branch out using the best split, providing the maximum information gain according to Shannon information theory, for each of the remaining attributes that the decision tree engine chooses as the most influential parameters.  When for a certain node the branches can no longer be split to provide a significant information gain for the branching nodes, the node is labeled a leaf node.  When a leaf node is established it is assigned a final decision.  The leaf node, highlighted in black, in the above diagram has the final value of “usa” where 33 records fit the criteria set by the previous branches and the origin of production of all the corresponding cars is USA.  Another important parameter in the right window is the significance, which indicates whether the final decision has any statistical significance. 

 

The Decision Tree engine also produces a rule that can be used for classifying future cases similarly to the Find Laws rule that you created earlier in this lesson.

 

Summarizing: We have seen how Linear Regression can be used to find dependencies in data, and how the powerful PolyNet Predictor and Find Laws algorithms provide better results when the dependencies are nonlinear in form.  Also, we have learned the method of applying the Decision Tree engine.
