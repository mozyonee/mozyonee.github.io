Tutorial 2: Medical Study

Brief Description: This tutorial incorporates the manipulation of datasets, graphing data for visualization, and using PolyAnalyst’s Rule Assistant.  The data in this tutorial is extracted from the archive of a large heart surgery clinic.  Each record corresponds to one patient.  The record fields, or attributes, are as follows: file number ID, sex, age, diagnosis, weight, and height.

Step 1: Import the Data

1.      From within the PolyAnalyst program, in the toolbar, click the  button to open a new project.  The file in this tutorial is labeled 2med.csv.  Its default location is: 

C:\Program Files\Megaputer Intelligence\PolyAnalyst 4.1\Examples\2med.csv

2.      Continue through the import wizard as explained in Tutorial 1 until the Edit Project dialog window.  In this tutorial there are more than two simple number values.  The fields in this CSV file are of different data types.

3.      Upon reaching the Edit Project dialog window, notice that PolyAnalyst has predetermined that the field DIAGNOSIS is a string.  There are some variables that need a more accurate data type so that PolyAnalyst can better work with the data.

4.      The sex field is “male” or “female”.  In technical terms this is a Boolean data type (also called logical, true/false, yes/no), a field that takes on one of two values that are opposite each other.  Check the box marked “Assume columns with 1’s and 0’s as ‘yes/no’ ”.

5.      After browsing through the CSV file notice that there are only five different sets of strings within the DIAGNOSIS field:  IHD (ishemic heart disease), MVD (mitral valve disease), AVD (arterial valve disease), TF (tetrada Fallo) and ASD (atrium septum defect).  DIAGNOSIS is of a category type.  Leave this field as string.  Later in this tutorial PolyAnalyst will determine that it is a category.

6.      Click OK.

PolyAnalyst imports the CSV file 2med.csv.  Double click on the world dataset to view this file from PolyAnalyst.

Step 3: Splitting Data

In this tutorial, looking at the different categories of patients, we see that we must divide up what we are studying because their medical characteristics may overlap and that one category will differ greatly than another category in its symptoms and patient characteristics.

PolyAnalyst provides the functionality to segment a data set.  In this tutorial, the World data set contains all the records of all of five categories of diagnosis.  We can segment the World data set into five diagnosis-specific data sets, in a way sorting out the information for further exploration.

1.      Right click on the World data set.  Select Split | To equal intervals… from the context menu.

2.      The Split to Equal Intervals window appears.

3.      Select the attribute DIAGNOSIS as the key to split the data set in this case the World data set.

4.      Click OK.  Five new data sets appear in the Workspace panel under Datasets, one for each diagnosis.

5.      Each of the new data sets now only contains its specific diagnosis.  To check if this is true, double click on a data set in the Workspace panel and view the DIAGNOSIS field in the Results panel.

Step 4: Creating a Histogram to Analyze Data

PolyAnalyst provides the user the ability to view insights into the data through graphical output.  For example, it may be of interest to examine the age of patients with different diagnoses.

1.      From the main file menu, select Create Object | Create Histogram….

2.      The Create new Histogram window appears.  In this window the data sets are listed on the left and the and the attributes are listed on the left.

3.      Select all the data sets by clicking on the data set name. Do not select World data set as it is cumulative and contains duplicate records of the diagnosis-specific data sets.

Age is a good attribute to measure the distribution of patient’s diagnosis.

4.      Select age as the X attribute by clicking on the word age.  Name the graph “Medical Histogram”.

5.      Click OK.  In the Workspace panel a new category called Illustrations and Graphs is created.  Under this category is the name of the histogram.  The graph is created in its own window in the Results panel.  There are several optional formatting functions you can explore in the right click context menu.

Interpreting the Histogram: This PolyAnalyst graphical output allows us to make a general inference about the data in question.  Looking at the dark blue “AVD Diagnosis” it is more prevalent among the lower age intervals and declines as age increases.  The opposite is true for the “MVD Diagnosis,” which increases with age.   AVD patients are usually children or young people while IHD and MVD patients are usually 40 years or older.

Step 5: Creating a Two Dimensional Graph to Test a Hypothesis

Doctors often wish to know if two diseases or patients symptoms are related in some manner.  Suppose the hypothesis that there is dependence between the height and weight of patients with ASD and patients with IHD.  A simple way to test for this dependence is through the creation of a two-dimensional chart comparing height and weight.

1.      From the main file menu, select Create Object | Create 2D-chart…  The Create new 2-D Chart window appears.

2.      Name the graph “Medical 2-D Chart”.

3.      To specify the attributes on the y-axis, the user must select attributes from the data sets in question.  To add a graph, click the button Add Data Graph.  A new dialog box appears named “Add object to view as a graph”.

4.      From the “Add object…” dialog first select DIAGNOSIS_“asd” and then its weight attribute.  Click OK.

5.      Repeat steps 3 and 4 for DIAGNOSIS_“ihd”(weight).

6.      Both weight attributes should now appear in the Graphs list in the Create 2-D chart window.

7.      To specify the attribute on the y-axis, select from the X-Attribute list the height attribute.  Your graph should now look like this:

8.      Click OK to create the chart.

For patients around 165 inches tall, the IHD patient coordinates appear clustered around 60 weight units while the ASD patients are clustered around 75 wieght units.  This graphical comparison suggests that IHD patients are generally heavier by 10 – weight units than ASD patients of similar height.

Step 6: Using Rule Assistant to Enhance Linear Regression

Suppose the hypothesis that a patient’s weight is dependant on the patient’s height and age.  Normally a statistician might try the quick method of discovering relationships using linear regression.  But linear regression can only discover linear relationships.  Linear regression does not serve as a good measure of the dependence in the hypothesis as weight is more proportional to the cube of height, rather than a direct linear proportion to height.

In order to use linear regression in this case, it makes sense to add the derivative independent parameter cube-of-height in order to simplify the exploration process.  PolyAnalyst provides the user the required functionality to add cube-of-height.

The first step is to calculate the cube-of-height using PolyAnalyst Rule Assistant for all records in the World data set.  PolyAnalyst uses a special feature of its own within the rule assistant called symbolic rule language.

1.      From within the main file menu, select Create Object | Create Rule…  The Create a new rule window appears.

2.      In the Name field enter the text “cube-of-height”.  This easily identifies the rule to you for later use.

About the Rule Assistant: the Rule Assistant provides a simple graphical user interface to the creation, editing, and viewing of rules.  The two main methods to create a rule in the Rule Assistant are to manually type the rule using PolyAnalyst’s symbolic rule language or to use the Rule Assistant’s buttons and rule builder system.

3.      Using the rule builder system, the first step is to create a function that will cube the attribute of height.

4.      Double-click the word Functions to open the functions list.  This will display a list of functions that PolyAnalyst uses in its rule creations.  Double click the word Math and then double click on pow to insert the Power function into the Rule box.

5.      Notice that the Rule text box now contains the function “pow(,)”.  The parentheses following pow represent the need for parameters in the pow function.  The comma within the parentheses specifies that the pow function requires two parameters, one before the comma and one after, in order to perform its function.  The first parameter is the original attribute.  The second parameter is the power by which to raise the first parameter.

6.      Click within the parentheses on the left side of the comma.  The original value is the variable height.  The Rule Assistant provides easy access to this variable through a variable list.  Double click the word Attributes, then double click the attribute height.  PolyAnalyst inserts the attribute into the rule.

7.      Move the cursor to the right side of the comma.  Enter the number 3, as this parameter is the power to which height is applied.

8.      Click OK.  PolyAnalyst adds the rule to the Workspace panel under the Rules section.  To later view or edit this rule, simply click the name in the Workspace panel.

9.      See Appendix A for a detailed discussion of the available functions you can use in rules.

Applying a Rule to a Data Set:  After creating a new rule, PolyAnalyst provides the user the easy functionality of drag and drop with the mouse to apply the rule to a specific set of data.  In this tutorial, when the cube-of-height rule is applied to the World dataset, it calculates the new value and creates a new field in the World data set that may be used for other exploration.

10.  Select the cube-of-height rule and drag it onto the World data set in the Workspace panel by releasing the click on top of the word World.  A special cursor will let you know what you are dragging.  Notice that now the World data set has two subsections, an attributes list and a rules list.

11.  Open the World data set.  The last field or column is now the applied rule cube-of-height.

Using the Linear Regression Exploration Engine:

12.  Right click on the World data set.  Select Explore | Linear Regression…(LR).  The Linear Regression on Dataset window appears.  Notice that PolyAnalyst also creates a linear regression rule that can be seen in the report on by clicking on the name of the rule in the Workspace panel.

13.  First select the target attribute weight by double clicking the word weight with the left mouse button.

14.  Deselect the attributes FILE and bsa by single clicking the words with the right mouse button.  These fields are not intuitively used in the prediction of weight.

15.  Click OK.  The linear regression exploration report appears in under the report section in the Workspace panel and in the Results panel.

About the Linear Regression Report: The report is comprised of a text section and a graphical output section.  In this sense it is similar to the Find Laws report described in Tutorial 1.

About the Text Section: The text report contains general information about the results of the performed exploration.  Notice that this exploration involved the attribute cube-of-height, and not height, in the calculated regression equation.  The statistical characteristics provide some additional insight.  Despite great confidence due to a large F-ratio, accuracy is low due to the large standard error of 33%.  This is not surprising as the exploration involved patients of different sex and age.  Ultimately, the exploration discovered that patients with MVD or IHD are on average heavier than other patients.  Looking at the positive coefficients of 4.509 and 13.86, one can see how the MVD/IHD patient is heavier by 4.5 and 13.86 kilograms respectively.  The ability to correctly include categorical attributes in exploration is one advantage of PolyAnalyst’s linear regression tool over several other statistical software packages.

About the Graph Section: This section of the linear regression report helps the user to visualize the results to gain a better insight and understanding of the data in question.  The gain chart will be discussed in a later tutorial.

Predicted versus Real Graph:  This is a basic scatter plot.  The closer the displayed coordinates lie to the diagonal line the better the accuracy of the prediction.

Predicted and Real versus Counter Graph: This graph simultaneously displays the predicted and real points per record to allow the user to view where the two values compare and contrast.

Residual Graph: The residual graph is also called the Cumulative distribution of errors.  It graphically displays a confidence interval.  A perfect or near perfect residual line will be in the shape of a slightly distorted “S”.  To illustrate this example, notice the coordinates (-10,5) and (10,90).  Looking at the coordinates on the y-axis, we see that there is a difference of about 85.  This is the 85% confidence interval.  If you expand the size of the Residuals graph, you can see that the calculated formula predicts weight with an error of less than 10kg in approximately 85% of cases.

16.  Close the report window.

About Context Menus: The PolyAnalyst interface takes full advantage of the right mouse button through the use of context menus.  To illustrate, right click on the linear regression rule of weight you just created under the rules section of the Workspace panel.  Select edit.  The rule assistant appears with the rule created during the exploration.  You can now edit this rule or cut and paste it into another statistical application like Microsoft Excel and test it out.  PolyAnalyst provides several functional advantages when intending to test data in several applications.

Step 7: Interacting with the 2-D chart

Suppose you wish to be able to test different values in a model you created.  PolyAnalyst provides the user the ability to plot found dependence on a graph.  This is a very convenient tool for visualization.

To do this we first edit the LR_weight rule.  Right click on the rule and select Edit.  Delete “cube-of-height” and replace it with pow(height,3).  (Don’t put the period).  Click OK.  The rule LR_weight now works directly with the data.

1.      From the main file menu in PolyAnalyst, select Create object | Create 2D-Chart.  The Create new 2D-Chart window appears.

2.      Click Add Rule Graph.  The Add object to view as graph window appears.

3.      Select the LR_weight rule.  Click OK.

4.      Select height as the X-attribute.  This will be displayed on the x-axis.

5.      Name the chart as 2D-Chart for later reference

6.      Click OK.  The 2D-Chart is created in the Illustrations and Graphs section of the Workspace panel and appears in the Results panel.

About the 2-D Chart: The created chart displays LR_weight as a function of height based on the attribute age, when age is fixed at 35.  Suppose you wish to test different values of the age attribute for each diagnosis.  PolyAnalyst provides the functionality to interact with and dynamically experiment with data.  The user accomplishes this through the use the slider or the drop down menu located on the right side of the chart.  (Note: if this section appears cut off please change your font settings to normal or small size in your control panel).  The graph is automatically updated and changes according to the changes in age and diagnosis.

More about the Workspace: Congratulations on successfully completing Tutorial Two.  At the end of the tutorial browse through the Workspace panel and see how it is organized into sections.  PolyAnalyst provides several right click context menus for quick and easy functionality.  Try right clicking on different elements in the tree list of the Workspace panel and notice which menus appear.

A good practice is to continue from what you learned in the last two tutorials and explore on your own other areas of PolyAnalyst.  Try and intelligently use a data exploration engine relevant to your data that successfully displays insight into the data.  Data mining in PolyAnalyst does require some knowledge of what type of data you are looking at and some goal of what you are looking for in the data, or to resolve from the data.  Two good exploration engines to try at this point are the Nearest Neighbor algorithm and or the Find Laws algorithm.  Compare the results of the reports.  PolyAnalyst’s suite of data mining tools provides the user an incredible amount of ability to derive knowledge from data to make decisions.
